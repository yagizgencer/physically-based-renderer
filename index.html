Physically-Based Renderer in C++

A physically-based renderer built on Nori, the minimalist C++ ray tracer of EPFL's CS-440 Advanced Computer Graphics course. Every feature below is implemented from scratch and statistically validated against Mitsuba 3, a research-grade renderer.

What is implemented

Light transport

Monte Carlo path tracing

Unbiased path tracing with next-event estimation, built on a modular object-oriented C++ architecture in which integrators, BSDFs, emitters and media are independently pluggable.

Appearance

Texture mapping

Image textures with bilinear interpolation and gamma correction, threaded through the BSDF eval() and sample() interfaces so any diffuse surface can be textured from the scene description.

Materials

Microfacet BSDFs

Rough conductor and rough dielectric models with microfacet distributions and a complex index-of-refraction Fresnel term for metals, following PBRT §9.6–9.7.

Illumination

Image-based lighting

HDR environment emitter importance-sampled by hierarchical sample warping over a mipmap of the light probe, giving low-variance estimates of high-contrast environments.

Volumetrics

Participating media

Volumetric path tracer with the Henyey–Greenstein phase function, spectral distance sampling, and beam transmittance carried along shadow rays so media attenuate direct lighting correctly.

Verification

Validated against Mitsuba 3

Every feature cross-checked at 256 samples per pixel against Mitsuba 3 reference renders, with per-pixel difference images and χ² tests confirming the sampling routines match their PDFs.

Technical report

Full derivations, implementation notes, side-by-side comparison sliders against Mitsuba 3, difference images and χ² test results for every feature.
